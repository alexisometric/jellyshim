Cache Headers

Add Cache-Control, ETag, and Vary headers to static files.
Cache duration for hashed/fingerprinted assets (e.g., main.a1b2c3d4.js). Default: 31536000 (1 year, immutable).
Cache duration for general static assets. Default: 2592000 (30 days).
Cache duration for community plugin static assets. Default: 86400 (1 day).
Serve stale content while revalidating in background. Default: 86400 (1 day).

Link / Preload Headers

Add Link: preload headers for font files to enable browser preloading.
Add Link: modulepreload headers for JavaScript files.

CORS / Resource Policy

Add CORP header (useful for iframe embedding, Jellyseerr integration).
cross-origin, same-site, or same-origin. Default: cross-origin.

Security Headers

Add X-Content-Type-Options, Referrer-Policy, and Permissions-Policy headers.

Image Optimization

✓ Built-in image processing

JellyShim processes images natively using ImageSharp + ffmpeg — no external service required.
Images are resized, re-encoded (AVIF/WebP/JPEG), and cached to disk automatically. AVIF uses Jellyfin's bundled ffmpeg (libaom-av1) — zero additional install. Each Jellyfin image type has independent width and quality settings.

● Checking AVIF support…

Intercept image requests and process them with resizing, quality reduction, and format conversion.
Cache processed images to disk for instant serving on subsequent requests.
Preferred output format: avif, webp, jpeg, or auto (auto negotiates AVIF → WebP → JPEG based on browser support).
Cache-Control max-age for image responses. Default: 2592000 (30 days).
Serve stale image while revalidating in background. Default: 604800 (7 days).

Plugin Static Asset Paths

URL path prefixes for community plugin static assets (e.g., /JellyTweaks/). These paths get plugin-level caching and compression headers.
Filename patterns (wildcard * supported) for web assets patched at runtime by File Transformation plugins (HSS, Custom Tabs, JellyfinEnhanced, etc.). These files are captured AFTER transformation, then compressed and cached. Note: When any patterns are configured, ALL .chunk.js and .bundle.js files are automatically bypassed from pre-built cache — no need to list every chunk filename. Clear the JellyShim cache after updating these plugins.

Advanced Security

Caution
warning
These settings can break your web client if misconfigured.

Test thoroughly after enabling each option. HSTS requires HTTPS. CSP must match your setup exactly.

Send Strict-Transport-Security header to enforce HTTPS. Only enable if your server uses HTTPS.
HSTS settings
Duration browsers should remember to use HTTPS. Default: 31536000 (1 year).
Apply HSTS to all subdomains as well.
Add CSP header to prevent XSS and data injection attacks. Test thoroughly before enabling — incorrect CSP can break the web client.
CSP directive
The full CSP directive string. The default is a safe baseline for Jellyfin.
Prevent clickjacking by controlling iframe embedding. Disable if using Jellyseerr in an iframe.
X-Frame-Options value
DENY, SAMEORIGIN, or ALLOW-FROM uri. Default: SAMEORIGIN.

Image Optimization

check_circle
Built-in image processing

JellyShim processes images natively using ImageSharp + ffmpeg — no external service required.
Images are resized, re-encoded (AVIF/WebP/JPEG), and cached to disk automatically. AVIF uses Jellyfin's bundled ffmpeg (libaom-av1) — zero additional install.

hourglass_empty
Checking AVIF support…

Intercept image requests and process them with resizing, quality reduction, and format conversion.
Cache processed images to disk for instant serving on subsequent requests.
Preferred output format: avif, webp, jpeg, or auto (auto negotiates AVIF → WebP → JPEG based on browser support).
Image cache settings
Cache-Control max-age for image responses. Default: 2592000 (30 days).
Serve stale image while revalidating in background. Default: 604800 (7 days).

Plugin Static Asset Paths

URL path prefixes for community plugin static assets (e.g., /JellyTweaks/). These paths get plugin-level caching and compression headers.
Filename patterns (wildcard * supported) for web assets patched at runtime by File Transformation plugins (HSS, Custom Tabs, JellyfinEnhanced, etc.). These files are captured AFTER transformation, then compressed and cached. Note: When any patterns are configured, ALL .chunk.js and .bundle.js files are automatically bypassed from pre-built cache. Clear the JellyShim cache after updating these plugins.
